# 实施计划：保存到活码界面功能

## 需求概述
在"图片"和"文档"导航链接位置添加"保存到活码界面"复选框，支持用户勾选后，在生成二维码时弹出创建文件夹对话框，将生成的二维码保存到活码文件夹中。

## 实施检查清单

### 1. 修改TopNav.vue组件
- 在navItems数组中为'image'和'file'项添加saveToLiveCode属性（布尔值，默认false）
- 在导航链接渲染处，为'image'和'file'添加复选框UI
- 添加复选框的点击事件处理，更新对应navItem的saveToLiveCode状态
- 通过emit向父组件传递saveToLiveCode状态变化事件

### 2. 修改App.vue组件
- 添加响应式变量保存图片和文档的"保存到活码"状态
- 监听TopNav的saveToLiveCode事件，更新对应状态
- 将状态通过props传递给QRGenerator组件

### 3. 修改QRGenerator.vue组件
- 添加props接收saveToLiveCode状态（image和file两个状态）
- 在generateQR函数中，当activeTab为'image'或'file'且对应saveToLiveCode为true时：
  - 生成二维码后，弹出创建文件夹对话框
  - 用户输入文件夹名称后，创建文件夹并保存到dynamicQRFolders
  - 将生成的二维码保存到dynamicQRList，关联folderId
- 创建saveQRToLiveCode工具函数，处理二维码保存逻辑
- 确保保存的二维码对象包含必要字段：id, title, fileName, data, folderId, status, scanCountToday, scanCountTotal, createTime, qrCodeUrl, qrUrl

### 4. 创建保存二维码到活码的工具函数
- 在QRGenerator.vue中创建saveQRToLiveCode函数
- 函数功能：
  - 接收二维码数据（dataURL, qrInfo, folderId, folderName）
  - 生成二维码对象，包含所有必要字段
  - 调用generateQRCode生成二维码图片
  - 保存到dynamicQRList
  - 更新文件夹的二维码数量

### 5. 创建文件夹对话框组件
- 在QRGenerator.vue中添加创建文件夹对话框的UI
- 对话框包含：输入框（文件夹名称）、取消按钮、确认按钮
- 添加响应式变量控制对话框显示/隐藏
- 实现创建文件夹逻辑，保存到dynamicQRFolders

### 6. 集成QRCode库用于生成二维码图片
- 确保QRGenerator.vue已导入QRCode库
- 在saveQRToLiveCode函数中使用QRCode.toDataURL生成二维码图片

### 7. 测试验证
- 测试图片标签页：勾选"保存到活码"，生成二维码，验证是否弹出文件夹创建对话框
- 测试文档标签页：勾选"保存到活码"，生成二维码，验证是否弹出文件夹创建对话框
- 验证生成的二维码是否正确保存到dynamicQRList
- 验证文件夹是否正确创建并保存到dynamicQRFolders
- 验证文件夹中的二维码数量是否正确更新
- 验证不影响其他功能（未勾选时正常生成二维码）

### 8. 最终动作
- 确保所有功能正常工作
- 确保不影响现有功能
- 代码审查，确保没有语法错误和逻辑错误
