# 排查"文件未找到"问题

## 快速检查清单

### 1. 检查 API 服务器是否运行

在服务器终端运行：

```bash
# 检查 PM2 服务
pm2 list

# 应该看到 qrcode-api 在运行
# 如果没看到，启动它：
cd /www/wwwroot/qrcode/2weima/api-server
pm2 start server.js --name qrcode-api
```

### 2. 测试 API 服务器

```bash
# 获取服务器IP
curl ifconfig.me

# 测试 API（替换为你的IP）
curl http://你的服务器IP:3000/health

# 应该返回：{"status":"ok","timestamp":...}
```

如果返回错误，说明：
- API服务器没有运行
- 端口没有开放
- 防火墙阻止了连接

### 3. 检查环境变量配置

```bash
cd /www/wwwroot/qrcode/2weima

# 查看环境变量文件
cat .env.production

# 应该看到：
# VITE_API_BASE_URL=http://你的服务器IP:3000
```

如果没有这个文件或内容不对，创建/编辑它：

```bash
nano .env.production
```

添加：
```env
VITE_API_BASE_URL=http://你的服务器IP:3000
```

### 4. 确认项目已重新构建

```bash
cd /www/wwwroot/qrcode/2weima

# 重新构建
npm run build

# 确认 dist 目录已更新
ls -la dist/
```

### 5. 检查浏览器控制台

在手机上打开二维码链接后：
1. 打开浏览器开发者工具（如果可能）
2. 或者用电脑访问同样的链接
3. 查看控制台（Console）的错误信息

常见错误：
- `Failed to fetch` - API服务器无法连接
- `404 Not Found` - API路径错误
- `CORS error` - 跨域问题

### 6. 检查 API 服务器日志

```bash
# 查看 API 服务器日志
pm2 logs qrcode-api

# 看看有没有请求记录
```

### 7. 手动测试保存和获取

在浏览器控制台运行（替换为你的服务器IP）：

```javascript
// 测试保存
fetch('http://你的服务器IP:3000/api/content', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    id: 'test123',
    fileName: 'test.txt',
    dataURL: 'data:text/plain;base64,dGVzdA=='
  })
}).then(r => r.json()).then(console.log)

// 测试获取
fetch('http://你的服务器IP:3000/api/content/test123')
  .then(r => r.json())
  .then(console.log)
```

## 常见问题解决

### 问题1：API服务器没有运行

```bash
cd /www/wwwroot/qrcode/2weima/api-server
pm2 start server.js --name qrcode-api
pm2 save
```

### 问题2：端口没有开放

```bash
# 开放3000端口
sudo ufw allow 3000
sudo ufw reload

# 检查端口是否监听
sudo netstat -tlnp | grep 3000
```

### 问题3：环境变量没有生效

```bash
# 确认文件存在
cat .env.production

# 重新构建
npm run build

# 重启 Nginx
sudo systemctl restart nginx
```

### 问题4：CORS 跨域问题

检查 `api-server/server.js` 中是否有 `cors()` 中间件。

### 问题5：API地址配置错误

确认 `.env.production` 中的地址是正确的：
- 格式：`http://IP:3000` 或 `http://域名:3000`
- 不要有尾随斜杠
- 使用 http 而不是 https（除非配置了SSL）

## 调试步骤

1. **在服务器上测试API**：
   ```bash
   curl http://localhost:3000/health
   ```

2. **从外部测试API**：
   ```bash
   curl http://你的服务器IP:3000/health
   ```

3. **检查数据是否保存**：
   ```bash
   ls -la /www/wwwroot/qrcode/2weima/api-server/data/
   ```

4. **查看API服务器日志**：
   ```bash
   pm2 logs qrcode-api --lines 50
   ```
