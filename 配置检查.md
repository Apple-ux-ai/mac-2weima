## 📝 需要你手动配置的步骤

### 1. 修改 API 地址

编辑 `.env.production` 文件，将 `VITE_API_BASE_URL` 修改为你的实际公网API地址：

```env
VITE_API_BASE_URL=https://your-api-domain.com
```

### 2. 或者在代码中直接配置

如果不想使用环境变量，可以编辑 `src/utils/api.js` 文件，修改第24行的默认值：

```javascript
return 'https://your-api-domain.com'  // 改为你的实际API地址
```

### 3. 配置 Viewer 页面的 API 地址（可选）

如果需要固定 viewer 页面的 API 地址，编辑 `public/advanced-viewer.html`，在 `<head>` 部分取消注释：

```html
<script>
  window.API_BASE_URL = 'https://your-api-domain.com'
</script>
```

## 🧪 测试步骤

### 1. 测试 API 连接

在浏览器控制台运行：

```javascript
// 测试保存
fetch('https://your-api-domain.com/api/content', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    id: 'test123',
    title: '测试',
    html: '<p>测试内容</p>'
  })
}).then(r => r.json()).then(console.log)

// 测试获取
fetch('https://your-api-domain.com/api/content/test123')
  .then(r => r.json())
  .then(console.log)
```

### 2. 重新构建

```bash
npm run build
```

### 3. 测试完整流程

1. 启动开发服务器：`npm run dev`
2. 在高级编辑器中编辑内容
3. 生成二维码

## 🔍 调试信息

- 打开浏览器控制台，查看是否有 API 请求错误
- viewer 页面会在控制台输出当前使用的 API 地址
- 如果 API 失败，会自动降级到本地存储

## ⚠️ 注意事项

1. **CORS配置**：确保你的API服务器允许跨域请求
2. **HTTPS**：生产环境必须使用 HTTPS
3. **API格式**：确保API返回格式符合要求（见 API配置说明.md）
